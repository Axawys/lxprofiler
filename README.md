# 🐧 Linux Psychological Profiler

Интерактивный профайлер на чистом **bash**, который «читает» вашу систему и
определяет психологический типаж пользователя. Скрипт анализирует дистрибутив,
окружение рабочего стола, железо, init-систему и установленные инструменты —
а затем распределяет очки между 12 архетипами и показывает результат в виде
интерактивного TUI.

## Установка

Одной командой:

```bash
curl -fsSL https://raw.githubusercontent.com/Axawys/lxprofiler/main/install.sh | bash
```

Скрипт клонирует репозиторий в `~/.local/share/lxprofiler` и создаёт команду
`lxprofile` в `~/.local/bin`, а также короткую команду `lx` — если она не занята
другой командой или алиасом. После этого профайлер запускается из любого места:

```bash
lxprofile   # или короткое: lx
```

> Запуск чужих скриптов из сети — на ваш страх и риск; при желании сначала
> просмотрите содержимое:
> `curl -fsSL https://raw.githubusercontent.com/Axawys/lxprofiler/main/install.sh | less`.

Нужен `git`. Если `~/.local/bin` не в `$PATH`, установщик подскажет, что добавить.

### Команды

| Команда | Сокращение | Действие |
|---------|------------|----------|
| `lxprofile` | | запустить профайлер |
| `lxprofile --static` | `-s` | статическая сводка вместо интерактива |
| `lxprofile --update` | `-u` | обновить утилиту с GitHub |
| `lxprofile --remove` | `--rm` | удалить утилиту из системы |
| `lxprofile --version` | `-v` | показать версию |
| `lxprofile --help` | `-h` | краткая справка |

Флаги принимаются в трёх равнозначных формах: с дефисом (`lx -u`), без дефиса
(`lx u`, `lx static`) и слитно с короткой командой `lx` — `lxu`, `lxs`, `lxv`,
`lxh`, `lxc`, `lxrm`. Слитные команды создаются при установке/обновлении, если
их имена свободны.

`--remove` снимает симлинк `lxprofile` и удаляет состояние; каталог установки
стирается только если это стандартная установка `install.sh`
(`~/.local/share/lxprofiler`) — dev-чекаут репозитория не трогается. Перед
удалением запрашивается подтверждение.

### Автообновление

При запуске в терминале `lxprofile` сверяет свою версию с репозиторием на
GitHub **в фоне** — утилита открывается сразу, не дожидаясь сети. Если вышла
новая версия, предложение обновиться появляется **после закрытия просмотра**:

```
Доступна новая версия lxprofile: 4.3.0 (у вас 4.2.0).
Обновить сейчас? [y/N]
```

- `y` — обновится и сразу перезапустится на новой версии;
- `n` — больше не будет предлагать, пока вы не обновитесь вручную
  (`lxprofile --update`); после ручного обновления подсказки снова включаются
  для будущих релизов.

Вместе с предложением показывается список изменений (`CHANGELOG.md`) — всё новое
из всех версий между вашей и доступной.

Проверка идёт с коротким таймаутом и не мешает работе оффлайн; в пайпе и не-TTY
она не выполняется.

### Без установки

Можно просто склонировать репозиторий и запустить локально:

```bash
git clone https://github.com/Axawys/lxprofiler
cd lxprofiler
./lxprofile
```

## Возможности

- **12 архетипов** с прозрачным обоснованием — видно, *что именно* повлияло на оценку
- **Интерактивный TUI**: статичный список + зелёный маркер, который двигается по классам, и панель деталей внизу
- **Три режима** (`m`): список архетипов, линуксоидные координаты и забавная статистика истории команд
- Навигация стрелками `↑/↓` и в стиле vim `j/k`, прыжки `g`/`G`, выход по `q`
- Нормализованные баллы и прогресс-бары
- **Градуированные описания** — текст зависит от процента наполнения класса (0–20 / 21–50 / 51–75 / 76–100 %)
- **Поведенческий анализ** — учитывает реальное использование команд из `.bashrc`/`.zshrc` и истории, а не только наличие пакетов
- **Калибровка под дефолты** — то, что предустановлено в дистрибутиве (systemd, firewall, python…), не накручивает баллы
- **Без зависимостей** — только `bash 4+`, coreutils и `tput`
- Учёт импортозамещённого ПО
- Фолбэк на статический вывод при запуске в пайпе / не-TTY

## Что анализируется

- **Дистрибутив** (`/etc/os-release`) — Arch, Fedora, Debian, NixOS, Gentoo,
  Kali, Silverblue, Vanilla OS, Astra, RED OS, ALT (в т.ч. ALT Atomic), ROSA и десятки других
- **DE / WM** — KDE, GNOME, XFCE, Hyprland, sway, i3, niri, dwm, qtile, xmonad…
- **Тип сессии** — Wayland / X11 / TTY
- **Железо** — объём RAM, число ядер CPU
- **Init-система** — systemd, runit, OpenRC, s6, dinit
- **Загрузчик** — GRUB (с учётом кастомной темы), systemd-boot, rEFInd, Limine, syslinux/extlinux, EFISTUB, LILO
- **Состояние системы** — аптайм, возраст установки, число пакетов, чужие пакеты
  (AUR), кастомное ядро (zen/xanmod/liquorix/hardened/rt/lts), снапшоты ФС
  (snapper/Timeshift), ZFS, Nix/home-manager
- **Установленные инструменты** — контейнеры, языки, редакторы, БД, серверы,
  игровые лаунчеры, средства шифрования и аудита, пакетные менеджеры
- **Flatpak-приложения** — количество и категории (игры, dev, приватность, медиа)
- **Конфигурация пользователя** — shell, терминал, кастомные конфиги (ricing),
  шифрование диска, SSH-ключи, git-репозитории в `$HOME`, локаль
- **Поведение** — реально используемые команды из `.bashrc`/`.zshrc` и истории
  (`.bash_history`, `.zsh_history`, fish): docker, kubectl, ssh, git, nmap, flatpak…

## Запуск

```bash
lxprofile
```

Для статического вывода (например, чтобы сохранить результат) используйте пайп:

```bash
lxprofile | cat
lxprofile | tee profile.txt
```

## Управление в интерактивном режиме

| Клавиша | Действие |
|---------|----------|
| `↑` / `k` | листать вверх |
| `↓` / `j` | листать вниз |
| `→` / `l` | следующий режим (список → линуксоидные координаты → статистика) |
| `←` / `h` | предыдущий режим |
| `m` | следующий режим |
| `g` | к первому классу |
| `G` | к последнему классу |
| `q` | выход |

### Режимы

- **Список** — рейтинг архетипов с прогресс-барами и панелью «что повлияло».
- **Линуксоидные координаты** — координаты по аналогии с политическим компасом: горизонталь
  **новаторы ◄ ► традиции**, вертикаль **▲ контроль (всё руками) / ▼ удобство
  (из коробки)**. Одна точка `●` — твой центр масс. Квадранты: *Лаборатория*,
  *Цитадель Unix*, *Гладкое будущее*, *Тёплая гавань*.
- **Статистика** — забавная аналитика истории команд: любимая команда и как часто
  ты её вводишь, сколько раз запускал fastfetch и обновлялся, счётчики `sudo` и
  `rm -rf`, пойманные опечатки и «редактор-война» vim/nano/emacs.

## Требования

- `bash` 4.0+
- стандартные coreutils (`awk`, `grep`, `sort`, `find`, `df`…)
- `tput` (пакет `ncurses`) — для интерактивного режима

## Как это работает

Каждый признак системы начисляет очки одному или нескольким классам с коротким
обоснованием (`add <класс> <очки> "<причина>"`). При этом:

- **дефолты дистрибутива не накручивают баллы** — systemd, предустановленный
  firewall, SELinux/AppArmor, python и т.п. сами по себе не считаются сигналом;
- **наличие ≠ использование** — поверх установленных пакетов работает
  поведенческий анализ: команды из `.bashrc`/`.zshrc` и истории показывают, чем
  человек *реально* пользуется, и дают дополнительные очки при достижении порога.

После сбора данных баллы нормализуются относительно лидера (0–100 %), классы
сортируются по убыванию, а в нижней панели показывается описание выбранного
класса и список сработавших критериев. Описание выбирается по проценту наполнения
(четыре градации), поэтому класс с 10 % и класс со 100 % звучат по-разному.
Логику легко расширять — достаточно добавить ещё один вызов `add`.

## Структура проекта

```
lxprofile          точка входа: разбор --version/--update/--help и запуск
install.sh         установщик одной командой
lib/
  version.sh       номер версии
  data.sh          архетипы, описания, выбор текста по проценту
  helpers.sh       add()/has()/safe_*, инициализация счёта
  detect.sh        весь анализ системы, нормализация и сортировка
  compass.sh       линуксоидный компас (векторы и отрисовка)
  stats.sh         статистика истории команд
  render.sh        оформление, статический и интерактивный вывод
script.sh          обёртка обратной совместимости → lxprofile
```

Модули подключаются `lxprofile` по порядку: `detect.sh` опирается на `data.sh`
и `helpers.sh`. Чтобы добавить новый сигнал, достаточно дописать вызов `add` в
[lib/detect.sh](lib/detect.sh); новый архетип — строки в [lib/data.sh](lib/data.sh).
