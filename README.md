# 🐧 Linux Psychological Profiler

Интерактивный профайлер на **Go** ([Bubble Tea](https://github.com/charmbracelet/bubbletea)),
который «читает» вашу систему и определяет психологический типаж пользователя.
Анализирует дистрибутив, окружение рабочего стола, железо, init-систему,
загрузчик, установленные инструменты и историю команд — а затем распределяет
очки между архетипами и показывает результат в полноэкранном TUI.

> Раньше проект был написан на bash; начиная с 5.0 основная реализация — на Go.
> Историческая bash-версия сохранена в архивной ветке.

## Установка

Одной командой (нужен только `curl`, ни Go, ни git не требуются):

```bash
curl -fsSL https://raw.githubusercontent.com/Axawys/lxprofiler/main/install.sh | bash
```

Установщик скачивает готовый бинарник последнего релиза под вашу ОС/архитектуру
(linux amd64/arm64) в `~/.local/bin/lxprofile` и создаёт короткую команду `lx` —
если она не занята другой командой или алиасом. После этого:

```bash
lxprofile   # или короткое: lx
```

> Запуск чужих скриптов из сети — на ваш страх и риск; при желании сначала
> просмотрите содержимое:
> `curl -fsSL https://raw.githubusercontent.com/Axawys/lxprofiler/main/install.sh | less`.

Если `~/.local/bin` не в `$PATH`, установщик подскажет, что добавить.

### Команды

| Команда | Сокращение | Действие |
|---------|------------|----------|
| `lxprofile` | | запустить профайлер |
| `lxprofile --static` | `-s` | статическая сводка вместо интерактива |
| `lxprofile --update [ВЕР]` | `-u` | обновить с GitHub (или откатиться на версию ВЕР) |
| `lxprofile --changes [ВЕР]` | `-c` | changelog указанной версии или 5 последних |
| `lxprofile --remove` | `--rm` | удалить утилиту из системы |
| `lxprofile --version` | `-v` | показать версию |
| `lxprofile --help` | `-h` | краткая справка |

Флаги принимаются в трёх равнозначных формах: с дефисом (`lx -u`), без дефиса
(`lx u`, `lx static`) и слитно с короткой командой `lx` — `lxu`, `lxs`, `lxv`,
`lxh`, `lxc`, `lxrm`. Слитные команды создаются при установке/обновлении, если
их имена свободны.

`--remove` удаляет бинарник, наши короткие симлинки и каталог состояния
(`~/.local/state/lxprofiler`). Перед удалением запрашивается подтверждение.

### Обновление и откат

Обновления доставляются готовыми бинарниками через **GitHub Releases**:

```bash
lxprofile --update          # до последней версии
lxprofile --update 5.0.0    # откат/фиксация на конкретную версию
```

`--update` скачивает бинарник нужной версии под вашу архитектуру и атомарно
заменяет себя на месте.

При запуске в терминале утилита в фоне сверяет свою версию с последним релизом.
Если вышла новее — предложение обновиться появляется **после закрытия просмотра**:

```
Доступна новая версия: 5.0.0 -> 5.1.0
Что нового:
  5.1.0:
    - …
Обновить сейчас? [y/N]
```

- `y` — обновится сразу;
- `n` — больше не будет предлагать, пока вы не обновитесь вручную; после ручного
  `--update` подсказки снова включаются.

Проверка идёт с коротким таймаутом и не мешает работе оффлайн; в пайпе и не-TTY
она не выполняется.

### Сборка из исходников

Нужен Go (версия из `go.mod`):

```bash
git clone https://github.com/Axawys/lxprofiler
cd lxprofiler
go build -o lxprofile ./cmd/lxprofile
./lxprofile
```

### Релизы (для мейнтейнера)

Версия хранится в `Version` в [cmd/lxprofile/main.go](cmd/lxprofile/main.go) и на
релизе подставляется из тега. Публикация автоматическая: пуш тега `vX.Y.Z`
запускает GitHub Actions ([.github/workflows/release.yml](.github/workflows/release.yml)),
который собирает бинарники под linux amd64/arm64 и создаёт релиз. После этого
`--update` и `install.sh` их видят.

```bash
git tag v5.1.0 && git push origin v5.1.0
```

## Возможности

- **Полноэкранный TUI** (alt-screen, как htop): статичный список архетипов,
  зелёный маркер и панель деталей внизу; при ресайзе не видно прошлых команд.
- **Три режима** (`m` / ←→): список архетипов, линуксоидные координаты и забавная
  статистика истории команд.
- Навигация стрелками `↑/↓` и в стиле vim `j/k`, прыжки `g`/`G`, выход по `q`.
- **Градуированные описания** — текст зависит от процента наполнения класса.
- **Поведенческий анализ** — учитывает реальное использование команд из
  истории и rc-файлов, а не только наличие пакетов.
- **Калибровка под дефолты** — предустановленное в дистрибутиве не накручивает баллы.
- **Скрытые и таинственные классы** — появляются только при срабатывании.
- Фолбэк на статический вывод в пайпе / не-TTY.

## Архетипы

Базовые классы (видны всегда):

| Класс | Кого описывает |
|-------|----------------|
| **DevOps** | Контейнеры, k8s, Terraform, IaC — *cattle, не pets* |
| **Программист** | Языки, компиляторы, редакторы, IDE |
| **Сис-админ** | Живые серверы, демоны, бэкапы, мониторинг — *pets, не cattle* |
| **Минималист** | Мало процессов, лёгкие WM, dash, Alpine |
| **Последователь Столлмана** | Был здесь до systemd: Emacs, suckless, alt-init, Gentoo |
| **Райсер** | Вылизанный рабочий стол, кастомные конфиги, тема под скриншот |
| **Геймер** | Steam, Proton, Wine, библиотека игр |
| **Анонимус** | Tor, VPN, VeraCrypt, шифрование диска |
| **Хацкер** | nmap, Metasploit, Wireshark, инструменты аудита |
| **Импортозаместитель** | Astra/RED OS/ALT/ROSA/Calculate, 1С, Яндекс.Браузер, nginx |
| **Свидетель свежего ПО** | Wayland, Rust, rolling-release, COSMIC |
| **Атомарник** | NixOS, Silverblue, Vanilla OS, Flatpak, distrobox, ostree |
| **Творческая снежинка** | Фото/видео-редакторы и написание битов |

### Скрытые классы

Появляются только при заполненности больше 20 % — например: **Нормис**,
**Любитель легаси**, **Вайбкодер**, **Тайлинг**, **Neovim-аддикт**,
**Shell-коллекционер**, **Самосборщик**, **Пакетоман**, **Музыкант**,
**Фотограф**, **Видеомонтажёр**, **3D-моделлер**, **Писатель**, **Стример**,
**Ветеран**, **Тинкерер**, **Embedded-разработчик** и другие.

### Таинственные классы

Показываются со «сломанной» полоской и скрытым именем (открывается при наведении):
**ЭТО ВИРТУАЛКА!**, **WSL**, **АндроЕд**, **Дуалбут**, **Корпорат**.

## Управление в интерактивном режиме

| Клавиша | Действие |
|---------|----------|
| `↑` / `k` | листать вверх |
| `↓` / `j` | листать вниз |
| `→` / `l` | следующий режим (список → линуксоидные координаты → статистика) |
| `←` / `h` | предыдущий режим |
| `m` | следующий режим |
| `g` | к первому классу |
| `G` | к последнему классу |
| `q` | выход |

## Как это работает

Каждый признак системы начисляет очки одному или нескольким классам с коротким
обоснованием (`Add(<класс>, <очки>, "<причина>")`). Дефолты дистрибутива не
накручивают баллы; поверх наличия пакетов работает поведенческий анализ по
истории команд и rc-файлам. Затем баллы нормализуются относительно лидера
(0–100 %), классы сортируются по убыванию, а в нижней панели показывается
описание выбранного класса (одно из четырёх — по проценту наполнения) и список
сработавших критериев.

## Структура проекта

```
cmd/lxprofile/
  main.go            точка входа: разбор флагов, static/TUI, справка
  update.go          обновление/откат через GitHub Releases, автопроверка
  changelog.go       разбор встроенного CHANGELOG.md (-c/--changes)
  state.go           каталог состояния, короткие команды lx/lxu/…
  remove.go          удаление (--rm/--remove)
internal/
  data/              архетипы, описания, выбор текста по проценту
  detect/            анализ системы, нормализация, сортировка
  tui/               Bubble Tea: список, координаты, статистика, отрисовка
embed.go             встраивание CHANGELOG.md в бинарник
install.sh           установщик одной командой (скачивание релиза)
docs/                заметки по разработке и расчёту категорий
.github/workflows/   автосборка релизов по тегу
```
